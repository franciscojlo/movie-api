Movie API Documentation

API Endpoints

Business Logic URL HTTP Method Request Data Format Response Data Body Format
Return a list of ALL movies to the user /movies GET N/A JSON Array (e.g., [{ movie1 }, { movie2 }, ...])
Return data (description, genre, director, image URL, whether it's featured or not) about a single movie by title to the user /movies/:title GET N/A JSON Object (e.g., { movie data })
Return data about a genre (description) by name/title (e.g., “Thriller”) /genres/:name GET N/A JSON Object (e.g., { genre data })
Return data about a director ( birth year ) by name /directors/:name GET N/A JSON Object (e.g., { director data })
Allow new users to register /users POST JSON JSON Object (e.g., { user data })
Allow users to update their user info (username) /users/:userId PUT JSON JSON Object (e.g., { updated user data })
Allow users to add a movie to their list of favorites /users/:userId/favorites POST JSON Text (e.g., "Movie added to user favorites successfully")
Allow users to remove a movie from their list of favorites /users/:userId/favorites/:movieId DELETE N/A Text (e.g., "Movie removed from user favorites successfully")
Allow existing users to unregister /users/:userId DELETE N/A Text (e.g., "User with ID {userId} unregistered successfully")

Request Methods

GET: Used for retrieving data from the server.

POST: Used for creating new data on the server.

PUT: Used for updating existing data on the server.

DELETE: Used for deleting data from the server.

Make sure to use the appropriate request method for each API endpoint.
